转码第一天：从零开始搭建技术体系

一、为什么选择转码

最近开始认真思考职业方向，最终决定转向软件开发。

主要原因：

  1. 对技术更有兴趣，希望做偏逻辑和工程的工作
  2. 软件行业成长路径清晰，发展空间更大

二、当前基础情况

目前还在起步阶段：

  • 有基础编程接触（C/C++）
  • 对数据结构、操作系统有初步了解
  • 能阅读简单英文技术文档

👉 整体来说：还没有完整体系，也没有项目经验

三、今天做了什么

1. 搭建个人博客（Hexo + GitHub Pages）

  • 安装环境（Node.js / Git）
  • 初始化 Hexo
  • 配置 SSH
  • 成功部署上线

👉 第一次走通完整流程：本地 → 上线

2. 遇到的问题

  • SSH 连接失败（端口22问题）
  • 通过 443 端口解决
  • 修改配置不生效（理解 generate + deploy）

👉 收获：问题本身不难，关键是排查过程

四、接下来的学习计划

基础知识

  • 数据结构与算法
  • 操作系统
  • 计算机网络

技术方向

  • Python（基础 + 刷题）
  • Java（后端方向）

工具

  • Linux
  • Git

五、下一步

  • 每天学习数据结构
  • 开始刷算法题
  • 持续更新博客

六、总结

目前刚起步，但已经开始行动。

这个博客会记录：

  • 学了什么
  • 遇到什么问题
  • 怎么解决

希望之后回头看，能看到清晰的成长路径。
